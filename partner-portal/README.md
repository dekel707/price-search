# הזמנות איתן — פורטל פעיל ומבודד

זהו פורטל נפרד ופעיל עבור איתן. הוא קורא מחירון, לקוחות ושריונים מהמערכת הראשית דרך גשר שרתי קריאה בלבד, ושומר כל פעולה במסד PostgreSQL עצמאי לפני שהיא מסונכרנת בצורה חתומה למערכת הראשית.

## גבולות ההפרדה

- אין שימוש ישיר במסד הנתונים הראשי, ב-Blob הראשי או בנתוני המלאי מצד הדפדפן או מסד הפורטל.
- כל טבלת נתונים בפורטל מתחילה ב-partner_ ונמצאת במסד חדש שהוגדר רק עבור פרויקט איתן.
- נתוני המפרט מגיעים מגשר קטלוג ייעודי. המחירון, הלקוחות והשריונים מגיעים רק מגשר חתום לקריאה בלבד.
- הפורטל אינו כותב ישירות למסד או ל-Blob של המערכת הראשית. פעולות הזמנה, עריכה ומחיקה עוברות דרך נקודת קצה חתומה ונשמרות באופן אטומי ומגובה.
- כל הסודות נשארים בשרתים בלבד.

## הגדרה לפריסה

1. פרויקט Vercel זה משתמש בתיקיית המקור partner-portal.
2. הוא מחובר לריפו dekel707/price-search, ולכן כל פרסום ל-main יפרוס את הפורטל בנפרד.
3. מסד PostgreSQL נפרד מחובר רק לפורטל זה.
4. הפורטל נכנס עם קוד איתן המוגדר ב-Vercel.

## הגנות נתונים

- לפני ואחרי כל פעולה משנה נתונים נוצר צילום מצב מלא בטבלת partner_backups.
- המלאי, המחירון, הלקוחות והשריונים נקראים דרך הגשר החי; הקרנה אחרונה נשמרת במסד הפורטל לשחזור תצוגה במקרה של תקלה זמנית.
- בזמן הכנת הזמנה הפורטל רק מסמן את השריון; בזמן הסנכרון נבדקת יתרה עדכנית וההפחתה מתבצעת בשמירה אטומית ומגובה.
- סטטוס processing מונע לחיצה כפולה או יצירת הזמנה ראשית כפולה.
- אין בפורטל נתוני דוגמה או מסלול כניסה זמני.

אין לייבא מסד נתונים מהמערכת הראשית. הקריאה החיה נעשית רק דרך הגשר המצומצם והחתום.
